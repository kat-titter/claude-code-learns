as it goes

You explain a fix, it works — and by next session the agent has forgotten it ever happened. Brilliant, and amnesiac. This saves each hard-won lesson as a small Skill it reaches for on its own, so it stops re-making the same mistake. Built on what Claude already has (memory + Skills); the one new trick is a Skill whose only job is to write other Skills. The sort of thing that looks obvious about thirty seconds after you see it.

03 · a skill is just a folder + a SKILL.md  ·  example 1

---
name: macos-long-render
description: Wrap long Mac renders / video encodes / training in
  caffeinate so the Mac can't fall asleep and stall them.  <- when to use it
  # write this like the moment you'd reach for it. too vague = never used.
---
## the fix
caffeinate -dimsu python3 long_job.py
## how to spot it
# steady, then a sudden cliff overnight = it slept. (not heat.)

where it came from: "17s/frame seems like a lot" — a real moment, mid-render.

04 · example 2  ·  a skill can carry its own helper

  Check each source is real; keep the numbers lined up with the list.
---
## the trap
# a 2nd numbered list in HTML restarts at 1 → footnote [33] points wrong.
## the fix
one list, or set start= on the next one. then run the checker:
python3 ${CLAUDE_SKILL_DIR}/scripts/check-citations.py file.html

where it came from: "there was a citation problem." Its checker later confirmed an older project was actually fine — in one command.

08 · why it works — how memory actually does it

Your brain doesn't record everything. It works in two passes. A fast part (the hippocampus) grabs what just happened. A slower part (the cortex) takes the bits that matter and rewrites them into steady, reusable knowledge — without scribbling over what's already there.1 AI agents are now built the same way.2 The match is simple: jotting a lesson down is the fast grab; turning it into a Skill is the slow, lasting copy. And the wiring is real — cells that fire together strengthen their link (Hebb's rule3, later measured directly4). A bug you keep hitting is that kind of repeated pattern; the Skill locks it in.

The trick is not saving everything. If you turned every little hiccup into its own Skill you'd drown in them — and your brain doesn't do that either. A memory sticks only if it repeats, or if it felt important.58 So jot things down cheaply, but only promote the lessons that keep coming back. If a new one fits something you already know, fold it in instead of starting fresh.6 Do the tidying later, in a quiet pass, not mid-task7 — and let skills you never use fade.9 Grab fast, keep slowly, stay light. Your agent already has the genius; this just adds the part of the brain that keeps it.

these aren't made-up examples

Both skills above came straight out of one normal work session — not a staged demo. "17s/frame seems like a lot" became macos-long-render. "there was a citation problem" became citations. The lessons were already in the conversation; the system just kept them instead of letting them slip away at the end. That's the whole point — you capture from the chat you're already having. (Revolutionary, we know: writing things down.)
